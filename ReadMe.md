# Система учета посещаемости студентов

Информационная система для автоматизации учета посещаемости студентов РТУ МИРЭА с использованием веб-технологий.

##  Описание

Система предназначена для автоматизации процесса фиксации и анализа посещаемости учебных занятий. Приложение решает проблемы ручного учета посещаемости, устраняя риск потери данных, ошибки при подсчете пропусков и сложность оперативного анализа статистики.

###  Основные возможности

- **Автоматизация учета** — централизованная платформа для внесения данных о посещаемости
- **Разграничение доступа** — четыре роли пользователей (Гость, Студент, Преподаватель, Администратор)
- **Аналитика и отчетность** — автоматический расчет процента посещаемости, сводные отчеты, визуализация данных
- **Интеграция с расписанием** — синхронизация с учебным расписанием
- **Журналирование действий** — фиксация всех действий пользователей в audit-логе
- **Экспорт данных** — формирование отчетов в форматах Excel и PDF
- **Адаптивный интерфейс** — работа на различных устройствах, темная/светлая тема

##  Технологический стек

### Backend
- **Node.js** (v18+) + **Express.js** — REST API
- **PostgreSQL** (v14+) — реляционная СУБД
- **JWT** — аутентификация и авторизация
- **bcryptjs** — хеширование паролей
- **ExcelJS** & **PDFKit** — генерация отчетов

### Frontend
- **Vanilla JavaScript** (ES6+) — без фреймворков
- **Chart.js** — визуализация данных
- **CSS3** — адаптивная верстка, CSS Variables

### Тестирование
- **Jest** + **Supertest** — автоматизированное тестирование backend и frontend

##  Установка и запуск

### Требования
- Node.js v18 или выше
- PostgreSQL v14 или выше
- npm или yarn

### Шаг 1: Клонирование репозитория
```bash
git clone https://github.com/kensh1nGG/attendance-system.git
cd attendance-system
```

### Шаг 2: Настройка Backend
```bash
cd backend

# Установка зависимостей
npm install

# Создание файла .env
cp .env.example .env

# Редактирование .env (укажите ваши параметры БД)
# DB_USER=postgres
# DB_HOST=localhost
# DB_NAME=attendance_db
# DB_PASSWORD=your_password
# DB_PORT=5432
# PORT=3000
# JWT_SECRET=your_secret_key

# Запуск сервера разработки
npm run dev

# Или для продакшена
npm start
```

##  API Endpoints

### Аутентификация
```
POST   /api/auth/register           # Регистрация нового пользователя
POST   /api/auth/login              # Вход в систему
POST   /api/auth/forgot-password    # Запрос сброса пароля
POST   /api/auth/reset-password     # Сброс пароля
GET    /api/auth/users/students     # Список студентов (для преподавателей/админов)
```

### Посещаемость
```
GET    /api/attendance              # Получить записи о посещаемости
POST   /api/attendance              # Отметить посещаемость
```

### Расписание
```
GET    /api/schedule                # Получить расписание
POST   /api/schedule                # Создать занятие
GET    /api/schedule/:id/students   # Получить студентов группы для отметки
POST   /api/schedule/:id/attendance # Массовая отметка посещаемости
```

### Отчеты
```
GET    /api/reports/attendance-summary   # Сводка по посещаемости
GET    /api/reports/export/excel         # Экспорт в Excel
GET    /api/reports/export/pdf           # Экспорт в PDF
```

### Рейтинги
```
GET    /api/ratings                 # Получить рейтинг студентов
POST   /api/ratings/recalculate     # Пересчитать рейтинг (админ)
```

### Администрирование
```
POST   /api/admin/create-user       # Создать пользователя
```

### Уведомления
```
GET    /api/notifications/:user_id  # Получить уведомления пользователя
```

##  База данных

### Основные таблицы
- **users** — пользователи системы
- **attendance** — записи о посещаемости (с ограничением UNIQUE на student_id + date)
- **schedules** — расписание занятий
- **audit_logs** — журнал аудита действий
- **notifications** — система уведомлений
- **password_resets** — токены восстановления пароля

### Индексы
- `idx_attendance_student_date` — ускорение поиска по студенту и дате
- `idx_attendance_date` — ускорение аналитики по датам
- `idx_schedules_group` — ускорение поиска по группам

### Триггеры
- `trg_update_attendance_updated_at` — автообновление поля updated_at
- `trg_update_schedules_updated_at` — автообновление поля updated_at

##  Безопасность

Система соответствует требованиям **ГОСТ Р ИСО/МЭК 27001-2012**:

- **Хеширование паролей** — bcrypt с солью
- **JWT-аутентификация** — токены со сроком жизни 24 часа
- **Защита от XSS** — библиотека xss-clean, Helmet.js
- **Защита от SQL-инъекций** — параметризованные запросы
- **Rate Limiting** — ограничение количества запросов
- **CORS** — настройка разрешенных источников
- **Audit logging** — журналирование всех критических действий
